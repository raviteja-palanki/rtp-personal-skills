## The Framework

Seven levels, from least to most autonomous. The structural shift between Levels 4 and 5 is where code stops controlling and the model starts controlling.

| Level | Type | Who Decides | 2026 Example | Cost |
|:---:|:---|:---|:---|:---:|
| 1 | AI Feature | Code, always | Gmail Smart Compose, Grammarly | <$0.01 |
| 2 | Chatbot | Code, via decision trees | Supabase AI, support bots | $0.01–0.03 |
| 3 | AI Assistant | Human directs, AI responds | Perplexity search, ChatGPT | $0.01–0.05 |
| 4 | Copilot | Human decides, AI suggests | GitHub Copilot, Cursor | $0.01–0.10 |
| — | **⚡ Structural Shift** | **Code controls ↑ / Model controls ↓** | | |
| 5 | Agent | AI decides, within boundaries | Claude Cowork, Harvey AI, Notion AI | $0.10–0.50 |
| 6 | Autonomous Agent | AI decides, humans monitor | Claude Code, Devin 2.0, Codex | $0.50–$50+ |
| 7 | Multi-Agent | Distributed AI decisions | Codex App, Salesforce Agentforce | $1–$20,000+ |

**Core insight:** "Autonomy is a design decision, not a capability decision. Target the right level — not the highest."

**Most products span multiple levels.** GitHub Copilot is L1 (autocomplete), L3 (chat), and L6 (Workspace) simultaneously.

## When to Use

Designing any AI product, evaluating competitors, staffing engineering teams, setting user expectations, communicating to boards.

## When This Fails

- Pure advisory systems (read-only) — autonomy classification is overkill
- Rapidly evolving products where level changes weekly during beta
- Regulated environments where level is externally mandated
