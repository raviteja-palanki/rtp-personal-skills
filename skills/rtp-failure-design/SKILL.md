# Failure Design → MERGED INTO failure-modes

**Status:** This skill has been merged into `failure-modes` (v2.0) as of 5 APR 2026.

**Why:** failure-modes (WHAT breaks) and failure-design (HOW to handle it) are two halves of one workflow. Splitting them created a false boundary — you can't design failure UX without first mapping failure types, and you can't map failures without specifying the response. The merged skill flows: Identify → Quantify → Design Response → Verify.
