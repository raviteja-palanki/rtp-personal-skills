# UX Design Systems — 59 Company Reference Library

A curated collection of design system specifications from the world's best-designed products. Each file is a complete, AI-readable design reference — not a component library, but the *thinking* behind the components.

## What's inside each file

Every company file follows a 9-section standard:

1. **Visual Theme & Atmosphere** — The emotional intent. What the design *feels* like and why.
2. **Color Palette & Roles** — Every color with hex values, semantic roles, and usage context.
3. **Typography** — Font families, weights, sizes, line heights, and hierarchy rules.
4. **Component Stylings** — Buttons, inputs, cards, nav, modals — with specific CSS values.
5. **Layout & Spacing** — Grid systems, breakpoints, padding/margin scales, content width.
6. **Depth & Elevation** — Shadow systems, layering, z-index conventions, border treatments.
7. **Do's and Don'ts** — Explicit guardrails. What to avoid, what to embrace.
8. **Responsive Behavior** — How the system adapts across screen sizes.
9. **Agent Prompt Guide** — Pre-written prompt fragments for generating UI in this style.

## How to use

**Match a company's style:** Read the specific company file before generating UI. The Agent Prompt Guide section (section 9) has ready-made prompt fragments.

**Design exploration:** Compare how different companies solve the same problem. How does Stripe handle elevation vs. Linear? How does Tesla's typography compare to Apple's?

**Build a custom system:** Cherry-pick elements across companies. Stripe's gradient system + Linear's spacing scale + Notion's warm neutrals = a coherent custom system if you understand the principles behind each.

**Presentation design:** Use company palettes and typography rules when building slides that need to match a specific aesthetic.
