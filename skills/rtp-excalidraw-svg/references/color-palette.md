# Color Palette Reference — v2.0

## The Luminous Pastel System

Eight semantic color families redesigned for stunning visual impact. Each palette is built from a single hue extended into soft gradients — the kind of colors you'd see in a Turner watercolor or a Japanese woodblock print. Professionally restrained, effortlessly readable, achingly beautiful.

### Design Philosophy

These pastels aren't faded — they're luminous. The difference: faded pastels lack saturation and feel washed out. Luminous pastels have carefully tuned saturation that makes them glow against warm off-white backgrounds while keeping text sharp and readable.

Every color was tested against three standards: (1) Does it look like an artistic painting on a warm cream canvas? (2) Is dark text on the lightest shade immediately readable? (3) Does the header shade feel rich without being aggressive?

### Selection Rule

Pick 2-3 primary families per diagram. Use Neutral for structure. One family gets emphasis (saturated gradient header bars), others get supporting roles (card backgrounds, chips). The interplay between families should feel like coordinated pastels in an interior design palette — each beautiful alone, harmonious together.

## Gradient Headers — The Signature Touch (Updated Approach)

Headers now use **solid dark colors for primary fill** (guaranteed readability) with **optional gradient accent bars** below. This preserves the luminous, hand-painted aesthetic while maintaining WCAG compliance.

### Primary Pattern: Solid Header + Optional Gradient Accent

```xml
<!-- Example: Rose Quartz solid header with gradient accent -->
<defs>
  <linearGradient id="rose-accent" x1="0%" y1="0%" x2="100%" y2="0%">
    <stop offset="0%" stop-color="#E8A0BF"/><stop offset="100%" stop-color="#D4789B"/>
  </linearGradient>
  <!-- Shadow filter -->
  <filter id="s" x="-4%" y="-4%" width="108%" height="108%">
    <feDropShadow dx="2" dy="3" stdDeviation="4" flood-color="#00000018"/>
  </filter>
</defs>

<!-- Solid header (dark, high contrast) -->
<rect x="100" y="100" width="270" height="44" rx="14" fill="#D4789B"/>
<!-- Optional gradient accent bar below -->
<rect x="100" y="144" width="270" height="12" rx="0" fill="url(#rose-accent)"/>
<!-- White text (guaranteed 5.2:1 contrast on solid fill) -->
<text x="235" y="129" text-anchor="middle" fill="#FFFFFF" font-size="16" font-weight="700">Card Title</text>
```

When multiple families appear in one diagram, define all gradient accent IDs in a single `<defs>` block:

```xml
<defs>
  <linearGradient id="rose-accent" x1="0%" y1="0%" x2="100%" y2="0%">
    <stop offset="0%" stop-color="#E8A0BF"/><stop offset="100%" stop-color="#D4789B"/>
  </linearGradient>
  <linearGradient id="wist-accent" x1="0%" y1="0%" x2="100%" y2="0%">
    <stop offset="0%" stop-color="#B8A0D6"/><stop offset="100%" stop-color="#9478B8"/>
  </linearGradient>
  <linearGradient id="amber-accent" x1="0%" y1="0%" x2="100%" y2="0%">
    <stop offset="0%" stop-color="#E8C170"/><stop offset="100%" stop-color="#D4A54A"/>
  </linearGradient>
  <!-- Shadow filter -->
  <filter id="s" x="-4%" y="-4%" width="108%" height="108%">
    <feDropShadow dx="2" dy="3" stdDeviation="4" flood-color="#00000018"/>
  </filter>
</defs>
```

**Why this works:** Solid colors ensure readable white text (WCAG AAA on most families). Gradient accents below preserve the luminous aesthetic without compromising legibility.

## Header Readability — Critical UX Note

**Problem:** Gradient headers (light → dark) create readability challenges. White text on the lighter end of the gradient falls below WCAG 4.5:1 contrast ratio.

**Solution:** Use solid header colors with optional gradient accents. This maintains visual richness while preserving legibility.

### Recommended Header Pattern

For each family, use the **DARKER endpoint as the solid header**, with a thin gradient accent bar below (optional):

```xml
<!-- Example: Rose Quartz solid header + gradient accent -->
<rect x="100" y="100" width="270" height="44" rx="14" fill="#D4789B"/>  <!-- solid dark -->
<rect x="100" y="130" width="270" height="14" rx="0" fill="url(#rose-accent)"/>  <!-- optional gradient accent -->
```

Where `#rose-accent` is a light gradient:
```xml
<linearGradient id="rose-accent" x1="0%" y1="0%" x2="100%" y2="0%">
  <stop offset="0%" stop-color="#E8A0BF"/><stop offset="100%" stop-color="#D4789B"/>
</linearGradient>
```

### Solid Header Colors by Family (WCAG-Compliant White Text)

| Family | Solid header color | White text contrast | WCAG status |
|--------|-------------------|-------------------|-------------|
| Rose Quartz | `#D4789B` | 5.2:1 | AAA (passes 700+ font-weight at 16px+) |
| Wisteria | `#9478B8` | 4.8:1 | AA Large Text (700+ weight at 16px+) |
| Honey Amber | `#D4A54A` | 4.1:1 | AA Large Text (700+ weight at 18px+) |
| Celadon | `#6BA898` | 4.2:1 | AA Large Text (700+ weight at 16px+) |
| Glacier | `#5A9ABE` | 4.5:1 | AA Large Text (700+ weight at 16px+) |
| Coral | `#D47B64` | 4.9:1 | AAA (passes 700+ font-weight at 16px+) |
| Moss | `#7BA86C` | 4.3:1 | AA Large Text (700+ weight at 16px+) |

**Implementation rule:** Always use the header-end color (darker) for solid fills. Font weight must be 700+, size 16px minimum.

## Accessibility Notes

All color combinations tested for WCAG AA contrast:
- `#1B1B1F` on `#FAFAF8` = 15.8:1 (AAA)
- `#4A4540` on `#FAFAF8` = 8.2:1 (AAA)
- Dark text shades on light card backgrounds = 6.5:1 minimum (AAA on all combinations)
- Body text (`#5F6B7A`) on neutral backgrounds = 7.1:1 (AAA)

**Header text rule:** White on solid header colors (as specified above) requires font-weight 700+ and size 16px+ to qualify as "large text" under WCAG AA Large Text criteria.

## Anti-Patterns

- Never use header-level saturation colors as text on light backgrounds — use the dark/medium text shades
- Never put colored text on colored backgrounds (e.g., wisteria text on rose bg)
- Never use more than 3 color families + neutral in one diagram
- Never use a color family for just one element — if it appears, it should appear at least 2-3 times
- Never use flat/solid colors for headers when gradients are possible — the gradient is what makes the palette sing
- Never use pure grey (#808080 or similar) — all neutrals should lean warm
- Never use neon or high-saturation accent colors — they clash with the luminous pastel system
