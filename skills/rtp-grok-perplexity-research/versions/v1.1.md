# Research Synthesiser v1.1 — The Weekly Intelligence Orchestrator

## Purpose

This skill transforms raw research inputs (SuperGrok X signals + Perplexity Deep Research) into compounding intelligence across Ravi's three highest-leverage knowledge systems. It is the BRIDGE between "information collected" and "intelligence applied."

Without this skill, research accumulates but never compounds. With it, every week's signals become next week's sharper questions, updated course content, refined skills, and stronger interview answers.

### Step 3: Cross-Reference & Deduplicate

**Cross-reference matrix:**

```
For each significant finding from EITHER source:
├── Does the other source confirm it?      → HIGH CONFIDENCE
├── Does the other source contradict it?   → INVESTIGATE
├── Is it only from X?                     → SIGNAL (unverified)
├── Is it only from Perplexity?            → RESEARCH (may lack practitioner context)
└── Does it appear in 3+ separate files?   → PATTERN (candidate for rules.md)
```

**Deduplication protocol:**
Several SuperGrok schedules share accounts (e.g., @swyx appears in schedules 1, 2, 4; @bcherny in 3, 4, 7). When the same post appears across multiple schedule outputs:
- Count it as ONE finding, not multiple
- Note which schedules surfaced it (this indicates cross-domain relevance)
- The more schedules that independently flag something, the stronger the signal

### What Triggers a Skill Update

```
PATTERN CONFIRMED (3+ occurrences across different sources this week)
  → Add to relevant skill as a new rule or heuristic

PATTERN OBSERVED (1-2 occurrences, strong evidence)
  → Add to 5_Knowledge/hypotheses.md, tag with relevant skill name

NEW CAPABILITY DISCOVERED (Claude or tool feature that changes a skill's approach)
  → Flag for skill revision with specific recommendation

SKILL GAP IDENTIFIED (research reveals a need no current skill addresses)
  → Draft proposal for new skill, add to SKILL-LEARNING-LOG

EXISTING RULE CONTRADICTED (evidence against a confirmed rule)
  → Flag prominently — do NOT silently change. Demote from rules.md
    to hypotheses.md with contradicting evidence noted
```

### Skill Update Protocol

**CRITICAL:** Before ANY skill update:
1. Check `2_Skills/SKILL-REGISTRY.md` for current version
2. Copy current skill to `versions/v{current}.md`
3. Make the update to the main SKILL.md
4. Increment version in SKILL-REGISTRY.md
5. Log change in the skill's CHANGELOG

## Rules This Skill Follows

1. **Never fabricate findings.** If a week's research is thin, say so.

2. **Confirmed patterns only promote to rules.md.** 3+ independent occurrences across different sources. Not 3 tweets by the same person.

3. **Contradictions are flagged, never silently resolved.** Present both sides, ask Ravi to decide.

4. **Interview prep gets SPECIFICITY.** Every extraction must pass: "Would saying this make the interviewer think 'this person follows our work closely'?"

5. **Course content gets TEACHING TRANSFORMATION.** Apply at least one of Ravi's 10 Thinking Algorithms.

6. **Skill updates get VERSIONING.** Copy-old → update → increment → log. Non-negotiable.

7. **The synthesis report is the SINGLE SOURCE OF TRUTH** for this week's intelligence.

8. **Respect the zones.** READS from Notion + `3_Research/`. WRITES to `1_Projects/`. PROPOSES to `2_Skills/`. UPDATES `5_Knowledge/`.

9. **Dedup across schedules.** Same post in multiple schedule outputs = one finding, stronger signal.

10. **Prompt refinement is part of synthesis.** Every weekly synthesis includes schedule performance review with concrete prompt edits.
