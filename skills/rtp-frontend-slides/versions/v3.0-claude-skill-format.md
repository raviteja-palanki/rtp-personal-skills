# Frontend Slides Skill

Create zero-dependency, animation-rich HTML presentations that run entirely in the browser. This skill helps non-designers discover their preferred aesthetic through visual exploration ("show, don't tell"), then generates production-quality slide decks.

## Core Philosophy

1. **Zero Dependencies** — Single HTML files with inline CSS/JS. No npm, no build tools. A single HTML file will work in 10 years.
2. **Show, Don't Tell** — People don't know what they want until they see it. Generate visual previews, not abstract choices.
3. **Anti-AI-Slop Design** — Avoid generic AI aesthetics. Every presentation should feel custom-crafted, not template-generated. No purple gradients on white. No Inter/Roboto. No standard blue.
4. **Production Quality** — Well-commented, accessible, performant code.
5. **Viewport Fitting (NON-NEGOTIABLE)** — Every slide MUST fit exactly within one viewport. No scrolling within slides, ever.

## Phase 0: Detect Mode

Determine what the user wants:

- **Mode A: New Presentation** — Create slides from scratch. Go to Phase 1.
- **Mode B: PPT Conversion** — User has a .ppt/.pptx file. Go to Phase 4.
- **Mode C: Enhancement** — User has existing HTML slides to improve. Read the file, understand the structure, enhance in place.

## Phase 1: Content Discovery (New Presentations)

Before designing, understand the content.

### Step 1.1: Presentation Context

Gather three things:

**Purpose** — What is this for?
Options: Pitch deck (selling to investors/clients), Teaching/Tutorial (educational), Conference talk (event/keynote), Internal presentation (team updates/strategy)

**Length** — How many slides?
Options: Short (5–10), Medium (10–20), Long (20+)

**Content readiness** — Does the user have content?
Options: All content ready, Rough notes to organize, Topic only (need full outline)

If user has content, ask them to share it (text, bullet points, images).

### Step 1.2: Image Handling

Ask the user whether they have images/assets to include. If yes:

**Evaluate each image for:**
- Usability: Is it clear, relevant, and high enough quality?
- Content signal: What concept does it represent? (e.g., "chat_ui.png" = conversational interface feature)

**Co-design principle:** Images shape the slide structure from the start — this is NOT "plan slides then add images." It's designing around both text and visuals simultaneously.

- 3 usable product screenshots → plan 3 feature slides, each anchored by one screenshot
- 1 usable logo → title slide and/or closing slide
- 1 architecture diagram → dedicated "How It Works" slide
- 1 blurry/irrelevant image → exclude, with explanation

If user selected "No images" → skip image pipeline entirely. Use CSS-generated visuals (gradients, shapes, patterns, typography) for visual interest.

Present the evaluation and proposed slide outline to the user for confirmation before proceeding.

## Phase 2: Style Discovery (Visual Exploration)

**CRITICAL: This is the "show, don't tell" phase.** Most people can't articulate design preferences in words. Generate mini-previews and let them react.

### Step 2.1: Mood Selection

Ask: "What feeling should the audience have?"

Options (can pick up to 2):
- **Impressed/Confident** — Professional, trustworthy
- **Excited/Energized** — Innovative, bold, futuristic
- **Calm/Focused** — Clear, thoughtful, easy to follow
- **Inspired/Moved** — Emotional, storytelling, memorable

### Step 2.2: Generate 3 Style Previews

Based on mood, generate 3 distinct single-slide HTML previews showing typography, color palette, animation style, and overall aesthetic.

**Mood → Style Mapping:**

| Mood | Style Options |
|------|---------------|
| Impressed/Confident | Corporate Elegant, Dark Executive, Clean Minimal |
| Excited/Energized | Neon Cyber, Bold Gradients, Kinetic Motion |
| Calm/Focused | Paper & Ink, Soft Muted, Swiss Minimal |
| Inspired/Moved | Cinematic Dark, Warm Editorial, Atmospheric |

**NEVER use these generic patterns:**
- Purple gradients on white backgrounds
- Inter, Roboto, or system fonts as primaries
- Standard blue primary colors
- Predictable hero layouts
- Cookie-cutter card grids

**INSTEAD, use distinctive choices:**
- Unique font pairings from Fontshare or Google Fonts (Clash Display, Satoshi, Cormorant Garamond, DM Sans, Syne, Space Mono, Instrument Serif, Cabinet Grotesk, General Sans)
- Cohesive color themes with personality and atmosphere
- Atmospheric backgrounds (gradient meshes, subtle patterns, noise textures, depth layers)
- Signature animation moments that surprise

Save previews to `.claude-design/slide-previews/`:
```
.claude-design/slide-previews/
├── style-a.html
├── style-b.html
└── style-c.html
```

Each preview: self-contained, single title slide, animated, ~50–100 lines.

Present to user and ask which they prefer, what they like, and what they'd change. Option to "Mix elements" from multiple styles.

## Phase 3: Generate Presentation

### CRITICAL: Viewport Fitting Requirements

This is NON-NEGOTIABLE. Copy this entire CSS block into every presentation:

```css
/* VIEWPORT FITTING: MANDATORY */
html, body {
    height: 100%;
    overflow-x: hidden;
}
html {
    scroll-snap-type: y mandatory;
    scroll-behavior: smooth;
}
.slide {
    width: 100vw;
    height: 100vh;
    height: 100dvh; /* Dynamic viewport for mobile */
    overflow: hidden; /* CRITICAL: No overflow ever */
    scroll-snap-align: start;
    display: flex;
    flex-direction: column;
    position: relative;
}
.slide-content {
    flex: 1;
    display: flex;
    flex-direction: column;
    justify-content: center;
    max-height: 100%;
    overflow: hidden;
    padding: var(--slide-padding);
}
```

**Content limits per slide:**
- Title slides: 1 heading + 1 subtitle + optional logo
- Content slides: 1 heading + max 5 bullet points OR 1 heading + 1 paragraph + 1 visual
- Comparison/grid slides: max 4 cards
- Too much content? → SPLIT into multiple slides. Never scroll.

### Responsive Typography (MANDATORY)

ALL sizes must use clamp() for responsive scaling:

```css
:root {
    --title-size: clamp(1.5rem, 5vw, 4rem);
    --h2-size: clamp(1.25rem, 3.5vw, 2.5rem);
    --h3-size: clamp(1rem, 2.5vw, 1.75rem);
    --body-size: clamp(0.75rem, 1.5vw, 1.125rem);
    --small-size: clamp(0.65rem, 1vw, 0.875rem);
    --slide-padding: clamp(1rem, 4vw, 4rem);
    --content-gap: clamp(0.5rem, 2vw, 2rem);
    --element-gap: clamp(0.25rem, 1vw, 1rem);
}
```

### Height-Based Responsive Breakpoints

```css
@media (max-height: 700px) {
    :root {
        --slide-padding: clamp(0.75rem, 3vw, 2rem);
        --title-size: clamp(1.25rem, 4.5vw, 2.5rem);
    }
}
@media (max-height: 600px) {
    :root {
        --slide-padding: clamp(0.5rem, 2.5vw, 1.5rem);
        --title-size: clamp(1.1rem, 4vw, 2rem);
        --body-size: clamp(0.7rem, 1.2vw, 0.95rem);
    }

}
@media (max-height: 500px) {
    :root {
        --slide-padding: clamp(0.4rem, 2vw, 1rem);
        --title-size: clamp(1rem, 3.5vw, 1.5rem);
        --body-size: clamp(0.65rem, 1vw, 0.85rem);
    }
}
@media (max-width: 600px) {
    .grid { grid-template-columns: 1fr; }
    .nav-dots, .keyboard-hint { display: none; }
}
```

### CSS Gotcha

**Never negate CSS functions directly** — `-clamp()`, `-min()`, `-max()` are silently ignored by browsers with no console error. Always use `calc(-1 * clamp(...))` instead.

### HTML Architecture

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Presentation Title</title>
    <!-- Fonts (Fontshare or Google Fonts — never system defaults) -->
    <link rel="stylesheet" href="https://api.fontshare.com/v2/css?f[]=...">
    <style>
        /* === CSS CUSTOM PROPERTIES (THEME) === */
        :root {
            --bg-primary: #0a0f1c;
            --bg-secondary: #111827;
            --text-primary: #ffffff;
            --text-secondary: #9ca3af;
            --accent: #00ffcc;
            --accent-glow: rgba(0, 255, 204, 0.3);
            --font-display: 'Clash Display', sans-serif;
            --font-body: 'Satoshi', sans-serif;
            --slide-padding: clamp(2rem, 5vw, 4rem);
            --ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
            --duration-normal: 0.6s;
        }

        /* === VIEWPORT FITTING (MANDATORY) === */
        /* ... (see above) ... */

        /* === ANIMATIONS === */
        .reveal {
            opacity: 0;
            transform: translateY(30px);
            transition: opacity var(--duration-normal) var(--ease-out-expo),
                        transform var(--duration-normal) var(--ease-out-expo);
        }
        .slide.visible .reveal {
            opacity: 1;
            transform: translateY(0);
        }
        /* Stagger children */
        .reveal:nth-child(1) { transition-delay: 0.1s; }
        .reveal:nth-child(2) { transition-delay: 0.2s; }
        .reveal:nth-child(3) { transition-delay: 0.3s; }
        .reveal:nth-child(4) { transition-delay: 0.4s; }

        /* === ACCESSIBILITY: Reduced Motion === */
        @media (prefers-reduced-motion: reduce) {
            .reveal {
                transition: opacity 0.3s ease;
                transform: none !important;
            }
            .reveal:nth-child(n) { transition-delay: 0s; }
            canvas, .particle-bg { display: none; }
        }
    </style>
</head>
<body>
    <div class="progress-bar" role="progressbar" aria-label="Presentation progress"></div>
    <nav class="nav-dots" aria-label="Slide navigation">
        <!-- Generated by JS -->
    </nav>
    <section class="slide title-slide" aria-label="Title slide">
        <div class="slide-content">
            <h1 class="reveal">Presentation Title</h1>
            <p class="reveal">Subtitle or author</p>
        </div>
    </section>
    <section class="slide" aria-label="Slide 2">
        <div class="slide-content">
            <h2 class="reveal">Content Title</h2>
            <p class="reveal">Content body...</p>
        </div>
    </section>
    <script>
        /* === SLIDE PRESENTATION CONTROLLER === */
        class SlidePresentation {
            constructor() { /* ... */ }
            // Keyboard nav (arrows, space, escape)
            // Touch/swipe support
            // Mouse wheel navigation
            // Progress bar updates
            // Navigation dots
            // Intersection Observer for .visible class
        }
        new SlidePresentation();
    </script>
</body>
</html>
```

### Required JavaScript Features

Every presentation MUST include:

1. **SlidePresentation class** — Main controller with keyboard nav (arrows, space), touch/swipe, mouse wheel, progress bar, nav dots
2. **Intersection Observer** — Adds `.visible` class for scroll-triggered animations
3. **Keyboard hint** — Small overlay showing "Use arrow keys" that fades after 3 seconds

**Optional enhancements** (based on style):
- Custom cursor with trail effect
- Particle system canvas background
- Parallax depth layers
- 3D tilt on hover (cards)
- Magnetic button interactions
- Counter/number animations
- Text scramble/typewriter reveals

### Accessibility Requirements (MANDATORY)

Every presentation must include:

- Semantic HTML: `<section>`, `<nav>`, `<h1>`–`<h6>` in proper order
- `aria-label` on slides and navigation elements
- `role="progressbar"` on progress indicator
- Keyboard navigation that works completely without mouse
- `prefers-reduced-motion` media query that disables all motion, hides particle canvases, removes transition delays
- Sufficient color contrast (WCAG AA minimum: 4.5:1 for body text, 3:1 for large text)
- Focus indicators visible on all interactive elements
- Touch targets minimum 24x24 CSS pixels

### Code Quality Requirements

**Comments:** Every section gets clear comments explaining what it does, why, and how to modify it.

**Performance:**
- Use `will-change` sparingly (only on elements about to animate)
- Prefer `transform` and `opacity` animations (GPU-composited)
- Throttle scroll and mousemove handlers
- Disable heavy effects (particles, canvas, parallax) on mobile via `@media (max-width: 768px)`

## Phase 4: PPT Conversion

When converting PowerPoint files:

### Step 4.1: Extract Content

Use Python with `python-pptx`:

```python
from pptx import Presentation
from pptx.util import Inches, Pt
import json, os, base64

def extract_pptx(file_path, output_dir):
    prs = Presentation(file_path)
    slides_data = []
    assets_dir = os.path.join(output_dir, 'assets')
    os.makedirs(assets_dir, exist_ok=True)

    for slide_num, slide in enumerate(prs.slides):
        slide_data = {
            'number': slide_num + 1,
            'title': '',
            'content': [],
            'images': [],
            'notes': ''
        }
        for shape in slide.shapes:
            if shape.has_text_frame:
                if shape == slide.shapes.title:
                    slide_data['title'] = shape.text
                else:
                    slide_data['content'].append({
                        'type': 'text',
                        'content': shape.text
                    })
            if shape.shape_type == 13:  # Picture
                image = shape.image
                image_name = f"slide{slide_num+1}_img{len(slide_data['images'])+1}.{image.ext}"
                image_path = os.path.join(assets_dir, image_name)
                with open(image_path, 'wb') as f:
                    f.write(image.blob)
                slide_data['images'].append({
                    'path': f"assets/{image_name}",
                    'width': shape.width,
                    'height': shape.height
                })
        if slide.has_notes_slide:
            slide_data['notes'] = slide.notes_slide.notes_text_frame.text
        slides_data.append(slide_data)
    return slides_data
```

## Phase 5: Image Processing Pipeline

If the user provided images and a style has been chosen:

For each curated image:
1. Determine processing needs based on style (circular crop for logos, resize for large files)
2. Apply CSS framing (border, glow, shadow) for images whose colors clash with the style palette
3. Save processed images alongside originals in assets folder
4. Reference with relative paths in HTML

Dependency: Python Pillow library (`pip install Pillow`)

**CSS framing examples by style mood:**
- Neon Cyber: `border: 1px solid rgba(0, 255, 204, 0.3); box-shadow: 0 0 30px rgba(0, 255, 204, 0.1);`
- Midnight Executive: `border: 1px solid rgba(255, 255, 255, 0.08); box-shadow: 0 4px 24px rgba(0, 0, 0, 0.4);`
- Paper & Ink: `border: 2px solid #1a1a1a; box-shadow: 4px 4px 0 #1a1a1a;`

## Style Reference: Effect → Feeling Mapping

| Feeling | Animation | Typography | Colors | Effects |
|---------|-----------|-----------|--------|---------|
| **Dramatic/Cinematic** | Slow fades (1–1.5s), large scale transitions | Serif display + clean sans body | Dark BG, spotlight accents | Parallax, full-bleed images |
| **Techy/Futuristic** | Neon glows, glitch text, grid reveals | Mono accents + geometric sans | Cyan, magenta, electric blue | Particles, grid patterns, scanlines |

## Animation Patterns Reference

**Entrance animations:**
- Fade + Slide Up (default): `translateY(30px)` → `translateY(0)`
- Scale In: `scale(0.9)` → `scale(1)`
- Slide from Left: `translateX(-50px)` → `translateX(0)`
- Blur In: `filter: blur(10px)` → `filter: blur(0)`

**Background effects:**
- Gradient mesh: layered radial-gradients with transparency
- Noise texture: inline SVG data URI
- Grid pattern: linear-gradient repeating lines at 3% opacity
- Grain overlay: CSS `background-image` with tiny repeating noise

**Interactive effects (optional):**
- 3D tilt on hover using `perspective` + `rotateX`/`rotateY` on mousemove
- Magnetic buttons that subtly follow cursor proximity
- Counter animations using `requestAnimationFrame`
- Text scramble using random character cycling

## Troubleshooting

**Fonts not loading** — Check Fontshare/Google Fonts URL, ensure font names match in CSS.
**Animations not triggering** — Verify Intersection Observer is running, check `.visible` class.
**Scroll snap broken** — Ensure `scroll-snap-type: y mandatory` on html, each slide needs `scroll-snap-align: start`.
**Mobile issues** — Disable heavy effects at 768px breakpoint, test touch events, reduce particle count.
**Performance** — Use `will-change` sparingly, prefer `transform`/`opacity` animations, throttle event handlers.
**CSS clamp negation** — Never write `-clamp(...)`. Use `calc(-1 * clamp(...))` instead.
